git-Workflow

  • clonen

Um an einem Projekt weiter zu arbeiten, dass via Git in einem Repository gespeichert ist, ist es zuerst notwendig mit dem Befehl git clone das Repository auf den eigenen Rechner zu clonen

Das bedeutet, dass das Repository ohne Veränderungen kopiert wird (1:1 geklont).

  • neue Änderungen hinzuzufügen

Um dem Projekt neue Änderungen hinzuzufügen, wird der Befehl git add verwendet.

  • Zwischenspeicherung

Die Änderungen via git add werden nicht zwischengespeichert.

Eine Zwischenspeicherung der Änderungen wird erst mit dem Befehl git commit durchgeführt.

Die Zwischenspeicherung erfolgt in einer zentralen Datenbank (.git). Dieser Zwischenstatus kann später wieder aufgerufen werden.

  • Änderungen hochladen

Mit dem Befehl git push, in Verbindung mit dem Befehl "origin master", werden die gespeicherten Änderungen in das ursprüngliche Repository hochgeladen.

  • lokale Kopien aktualisiert

Über den Befehl git pull werden aktuelle Änderungen im Projekt geholt und lokale Kopien aktualisiert.
